## Permissions and Access Control

Manage user roles and access levels to secure your documentation.

<Tabs>
  <Tab title="Roles" icon="users">
    Define roles like `admin`, `editor`, and `viewer`.

    <ParamField path="permissions.role" param-type="string" required="true">
      User role: `admin` (full access), `editor` (edit docs), `viewer` (read-only).
    </ParamField>

    <ParamField path="permissions.invite_limit" param-type="number" required="false">
      Maximum invited users per role (`>0`).
    </ParamField>
  </Tab>
  <Tab title="API Access" icon="key">
    Generate tokens for programmatic access.

    <ParamField header="X-API-Key" param-type="string" required="true">
      Your API key for requests.
    </ParamField>
  </Tab>
</Tabs>

## Theme and Layout Options

Apply custom styles to reflect your brand.

Use this JSON configuration to update themes:

<CodeGroup tabs="JSON,YAML">
  ```json
  {
    "theme": {
      "primaryColor": "#3B82F6",
      "layout": "sidebar",
      "font": "Inter"
    }
  }
  ```
  ```yaml
  theme:
    primaryColor: "#3B82F6"
    layout: sidebar
    font: Inter
  ```
</CodeGroup>

<ParamField body="theme.primaryColor" param-type="string" required="false">
  Brand color, e.g., `#3B82F6`.
</ParamField>

## Integrating External Tools

Connect your space to GitHub, Slack, or webhooks for seamless workflows.

<Steps>
  <Step title="Add Integration" icon="plug">
    Navigate to Integrations in settings.
  </Step>
  <Step title="Configure Webhook" icon="settings">
````bash
curl -X POST https://api.example.com/v1/integrations \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "type": "webhook",
    "url": "https://your-webhook-url.com/webhook"
  }'
````
  </Step>
  <Step title="Test Connection" icon="check-circle">
    Send a test event from the dashboard.
  </Step>
</Steps>

## Backup and Export Settings

Protect your content with regular backups and exports.

<Expandable title="Advanced Backup Config" default-open="false">
  Schedule exports via API:

````javascript
const cron = require('node-cron');
cron.schedule('0 2 * * *', async () => {
  const response = await fetch('https://api.example.com/v1/spaces/{spaceId}/export', {
    method: 'POST',
    headers: { 'Authorization': 'Bearer YOUR_TOKEN' },
    body: JSON.stringify({ format: 'mdx' })
  });
});
````

</Expandable>

| Setting | Description | Default |
|---------|-------------|---------|
| `backup.frequency` | Cron schedule (e.g., `0 2 * * 0`) | Weekly |
| `export.format` | `mdx`, `pdf`, `html` | `mdx` |
| `storage.location` | S3 bucket or local path | Local |

<Callout kind="alert">
  Enable backups before inviting external collaborators.
</Callout>

## Troubleshooting Common Issues

- **Permission denied**: Verify your role with `GET /permissions`.
- **Theme not applying**: Clear cache and redeploy.
- **Integration failures**: Check webhook URL format and token scopes.
