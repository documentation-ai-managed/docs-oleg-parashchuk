## Upgrading Your Documentation

Follow these steps to upgrade to the latest version seamlessly.

<Steps>
  <Step title="Backup Your Data" icon="database">

    Export your current documentation:

````bash
npx oleg-docs export --project=my-project --format=zip
````

  </Step>
  <Step title="Update CLI" icon="download">

    Install the latest CLI version:

    <CodeGroup tabs="npm,yarn">
```bash
npm install -g @oleg/docs@latest
```
```bash
yarn global add @oleg/docs@latest
```
    </CodeGroup>

  </Step>
  <Step title="Run Migration" icon="settings">

    Apply migrations:

````bash
npx oleg-docs migrate --version=1.2.0
````

  </Step>
  <Step title="Verify" icon="check-circle">

    Test your site:

````bash
npx oleg-docs serve --project=my-project
````

  </Step>
</Steps>

<Callout kind="tip">
  For v1.0.0 users, update your API calls to use `{version: "v2"}` in headers. See migration notes below.
</Callout>

## Migration Notes

<ExpandableGroup>
  <Expandable title="From v1.0 to v1.2" default-open="true">

    Key changes include API base URL updates. Replace `https://api.example.com/v1/docs` with `https://api.example.com/v2/docs`.

    Example update:

````javascript
// Old
const response = await fetch('https://api.example.com/v1/docs', {
  headers: { Authorization: `Bearer ${YOUR_TOKEN}` }
});

// New
const response = await fetch('https://api.example.com/v2/docs', {
  headers: { 
    Authorization: `Bearer ${YOUR_TOKEN}`,
    'Api-Version': 'v2'
  }
});
````

  </Expandable>
  <Expandable title="Webhook Changes">

    New webhooks use `https://your-webhook-url.com/webhook`. Update your endpoint to handle `v2` payloads.

  </Expandable>
</ExpandableGroup>
